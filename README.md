# Анализ изменений процесса (NDA)

Во внутреннем процессе периодически возникали перегрузки в пиковые периоды. С 01.02 были внедрены новые правила работы, чтобы снизить накопление нагрузки и сделать процесс стабильнее.

В этом проекте я проверяю эффект изменений через сравнение периодов до/после. Все данные обезличены: используются индексы (среднее до изменений = 100), без публикации абсолютных значений.


## Что использовал

Две таблицы:
- дневной объём операций по датам;
- журнал перегрузок (дата, длительность, уровень нагрузки, комментарии).

## Как считал

- Граница “до/после”: 01.02.
- Индекс: среднее значение до изменений принимаю за 100.
- Чтобы убрать дневной шум, добавил 7-дневное сглаживание.

## Что получилось

После внедрения правил:
- общий индекс объёма снизился относительно периода “до”;
- перегрузки снизились более чем в 2 раза по индексу;
- пики стали проходить стабильнее.

## Что заметил по журналу перегрузок

- перегрузка чаще появляется из-за накопления очереди, а не одного всплеска;
- смешивание разных потоков в один момент усиливает нагрузку;
- эффект сильнее заметен именно в пиковые дни.

## Ограничения

Это сравнение “до/после” без контрольной группы. На динамику могли влиять внешние факторы (сезонность). Для строгого доказательства эффекта нужен контрольный дизайн, но для кейса важнее показать логику анализа и выводы.
